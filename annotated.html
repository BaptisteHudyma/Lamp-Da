 ►NalertsHandle the system alerts, that are displayed on the RGB indicator
 CAlert_BatteryCriticalThis alert is raised when the battery level falls below a critical threshold. Falling further will dammage the battery
 CAlert_BatteryLowThis alert is raised when the battery level gets low
 CAlert_BatteryMissingThis alert is raised when no connected battery is detected
 CAlert_BatteryReadingIncoherentThis alert is raised when the battery readings do not fit the battery physical model
 CAlert_BluetoothAdvertisementThis alert is raised when the Bluetooth advertising is turned on. It is just an informative alert
 CAlert_ChargerErrorThis alert is raised when the battery charger signals an error. It is not well handled at the moment, so it does not block anything
 CAlert_FavoriteSetThis alert is raised when new favorite mode is added
 CAlert_HardwareAlertThis alert is raised when a problem is detected in the hardware. Usually, it means that an electrical component may have become unresponsive
 CAlert_LongLoopUpdateThis alert is raised when the main loop becomes slower than the defined behavior. This is not a critical alert, but it sually indicates that the user mode in use is too slow
 CAlert_MainLoopFreezeThis alert is raised when the main loop becomes unresponsive
 CAlert_OtgFailedThis alert is raised when the output power fails to start. It can happen with logic bugs, or if the output rail is shorted
 CAlert_SkippedCleanSleepThis alert is raised when the system starts without sleeping cleanly first. It appears after updates, crashes, power failures, ..
 CAlert_SunsetTimerSetThis alert is raised when the sunset timer is active. It means that the system will turn off on it's own after a delay
 CAlert_SystemInErrorStateThis alert is raised when the system falls into an unrecoverable state. It can be raised by logic error, or detected bugs at program start
 CAlert_SystemInLockoutThis alert is raised when the user tries to power the system as it is in lockout mode
 CAlert_SystemShutdownFailedThis alert is raised when the system fails to go to sleep. There is no way to recover from this, as we depend on the low level handlers for this functionality
 CAlert_TempCriticalThis alert is raised when the temperature gets too high. If it keeps climbing, the components will get dammaged
 CAlert_TempTooHighThis alert is raised when the internal temperature gets high. If it keeps rising, the battery may get dammaged
 CAlert_UsbPortShortThis alert is raised when a short circuit is detected in the USB lines. The port should be cleaned before using it again
 CAlertBaseBase class for all alerts. Must be overloaded to define an alert
 CAlertManager_tHandle the whole alerts logic. Use this to raise and clear alerts, and make decisions based on those alerts. Alerts can be raised and cleared on their own
